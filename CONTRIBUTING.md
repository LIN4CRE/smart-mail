# Contributing to Smart Email

First off, thank you for considering contributing to Smart Email! It's people like you that make it such a great tool.

## Where do I go from here?

If you've noticed a bug or have a feature request, make sure to check our [Issues](https://github.com/your-username/smart-email/issues) first. If you don't see it, feel free to open a new issue.

## Pull Requests

1. Fork the repo and create your branch from \`main\`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes.
5. Make sure your code lints.
6. Issue that pull request!

## Code Style

- Use standard formatting for React and TypeScript (Prettier & ESLint are recommended).
- Ensure variables and functions are descriptively named.
